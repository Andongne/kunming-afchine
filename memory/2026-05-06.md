## Problèmes et leçons critiques

### Varnish (CDN Gandi)
- Cache les réponses PHP des scripts uploadés (404 ou vide) si curl intervient > 2-3s
- Technique fiable : upload SFTP + curl **immédiat** OU via sppb5.php `write_query`/`select_query`
- `purge_cache` via sppb5.php vide cache Joomla + OPcache + Varnish HTTP
- PURGE HTTP direct : `curl -X PURGE https://kunming-afchine.org/[url]`

### WAF Gandi (bloquages SQL via sppb5.php)
- Mots bloqués : `falang`, `XMLFile`, `ComponentName`, `ComponentType`, `Ordering`
- Fix : `UNHEX('hex')` pour les valeurs ; `INSERT SELECT` pour les volumes
- Apostrophes dans les valeurs SQL → aussi bloquées → UNHEX obligatoire

### FaLang
- Tables : `bwhwo_falang_content` et `bwhwo_falang_tableinfo` (PAS falang_translate)
- language_id=1=EN, language_id=4=ZH
- Colonnes : id, language_id, reference_id, reference_table, reference_field, value, original_value, published
- Mapper reference_ids par **alias** (menu.alias, content.alias) — IDs différents dev/prod

### Schéma DB RSForm (dev vs prod)
| dev | prod |
|---|---|
| XMLFile | ComponentTypeId |
| ComponentName | (dans rsform_properties.NAME) |
| Ordering | `Order` (mot réservé, backticks) |

### ModuleHelper::getModules()
- Peut retourner vide dans contexte override RSForm en prod
- Fix dans default.php : requête directe DB avec `menuid=-1 OR menuid=active_id`
- Prérequis : `menuid=-1` dans modules_menu + published=1 + access=1

### Menu nested set (lft/rgt)
- Items insérés avec lft=0, rgt=0 → invisibles dans la navigation
- Rebuild : `UPDATE ... SET rgt=rgt+2 WHERE rgt>=insert_pos` puis SET valeurs sur le nouvel item
- Toujours mapper par `alias`, jamais par `id` (IDs ≠ entre dev et prod)

### Problèmes rencontrés et résolus

**CGV checkbox spacing** (soirée 2026-05-06)
- Symptôme : espace insuffisant entre la case à cocher CGV et son texte
- Fix : règles CSS `.rsform-block-cgv label { display:flex; gap:10px; align-items:flex-start; }`
- Fichier : `afk-styles.css` + bump version `?v=2026050601` dans `index.php`

**Form 6 (cours) bloqué sur "champs manquants" en prod**
- Symptôme : formulaire refuse de soumettre même avec tous les champs remplis
- Cause 1 : anciens composants C87-C105 (16 composants) encore actifs avec REQUIRED=YES sur Form6 — validés en arrière-plan même si invisibles
- Cause 2 : propriété NAME absente sur Prénom (C110) et Email (C111) → rendu `{Prenom:caption}` etc. dans le layout
- Fix appliqué : masquage C87-C105 (Published=0), correction ComponentTypeId C109/C110/C111 (10→1), reinsertion NAME sur C110+C111, C125 type 10→13
- ⚠️ LEÇON : lors d'un déploiement Form6, vérifier systématiquement que TOUS les anciens composants sont masqués (Published=0) et que les textBox ont bien la propriété NAME

**Formulaire cours affiche `{Prenom:caption}` au lieu du label**
- Cause : composant Prenom (C110) avait ComponentTypeId=10 (freeText) au lieu de 1 (textBox)
- Fix : UPDATE bwhwo_rsform_properties SET ComponentTypeId=1 + ajout propriété NAME

**ModuleHelper retourne vide en prod**
- Fix permanent : fallback direct DB dans `default.php` RSForm override
- Code dans `html/com_rsform/rsform/default.php` ligne ~15
