### Problèmes rencontrés aujourd'hui (à éviter)
- Script construit après coup, pas avant → déploiements partiels en cascade
- Version JS dans index.php pas synchronisée sur dev → bug post-déploiement
- OPcache multi-worker : les modifications de fichiers PHP ne prennent effet qu'après `opcache_invalidate()` explicite sur le bon worker

### Règle à respecter
- Toujours bumper la version dans `index.php` sur DEV en même temps que la modification CSS/JS
- Tester sur dev.kunming-afchine.org avant tout déploiement
- Un seul déploiement via `deploy_dev_to_prod.php`, pas de corrections partielles
