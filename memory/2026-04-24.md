## Bug critique identifié — Script de traduction
Le script `translate-exam-en-final.py` corrompt les champs HTML contenant des guillemets échappés `\"`.
Lors de la recherche `fr_pattern` avec `content.find(...)`, le `end_q` se termine sur un `"` à l'intérieur d'un attribut HTML (`<p class=\"..."`), tronquant le FR et insérant l'EN au mauvais endroit.

**Fix à implémenter demain** : utiliser une recherche de `end_q` qui respecte les `\"` dans le contexte JSON (depth tracking ou regex avec lookahead négatif pour `\\"`).
