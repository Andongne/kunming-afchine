## Formulaire d'inscription aux examens (FormId=4, page id=239, record SPB 750)

### Problèmes résolus

#### 1. Popup de confirmation FR vide (nom/session manquants)
- **Cause** : le JS cherchait `form[id*="rsform"]` mais le formulaire a `id="userForm"` → event submit jamais capturé → sessionStorage vide
- **Fix** : sélecteur élargi à `form#userForm, form[id*="rsform"], form[name*="rsform"]` dans le champ JS de RSForm (bwhwo_rsform_forms.JS)

#### 2. Emails (admin + user) avec placeholders non résolus
- **Cause** : format `{Champ}` non reconnu — RSForm Pro attend `{Champ:value}`
- **Fix** : tous les placeholders convertis en `{Champ:value}` dans AdminEmailText, UserEmailText, AdminEmailSubject, UserEmailSubject
- `UserEmailTo` corrigé : était `accueil.kunming@afchine.org` → `{Email:value}`
- `AdminEmailMode` passé à 1 (HTML)

#### 3. Formulaire ZH — aucune session disponible
- **Cause** : traduction RSForm (bwhwo_rsform_translations id=225) traduisait les ITEMS de Choix_exam (ComponentId=54) avec des valeurs chinoises (`TCF 加拿大`) incompatibles avec les clés de `rse_by_category` (`TCF Canada`)
- **Fix** : format `valeur|libellé` → `TCF Canada|TCF 加拿大` etc.

#### 4. Dates affichées en français sur la page ZH
- **Fix** : `toFrench()` adapté — détecte `document.documentElement.lang` → format `2026年7月7日` en ZH

#### 5. Popup en français sur toutes les versions
- **Fix** : JS section A modifié pour détecter la langue et construire la popup appropriée :
  - `fr` → template `rsf-ty-template` existant
  - `zh` → popup construite inline en chinois
  - `en` → popup construite inline en anglais (prête pour quand la version EN est republiée)

#### 6. Sujets email ZH (ids 229, 231 dans bwhwo_rsform_translations)
- Étaient des messages RSForm par défaut en anglais
- Corrigés : `报名确认 — {Choix_exam:value}，{Session:value} | 昆明法语联盟` / `[报名] {Choix_exam:value} — {Session:value} — {Name:value} {Prenom:value}`

### État du Thankyou DB
- `bwhwo_rsform_forms.Thankyou` : modifié (placeholders `{Prenom}`, `{Name}`, `{Session}` sans `:value`) mais non utilisé (ShowThankyou=0) — neutre
- La popup est gérée entièrement par le JS custom (sessionStorage + overlay)
