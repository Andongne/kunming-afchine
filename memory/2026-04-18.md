### FaLang — Architecture et hreflang (documentation Faboba, 18 avril 2026)

**Fonctionnement de FaLang :**
- FaLang traduit le contenu en base à la même URL via remplacement DB (driver)
- Le switch de langue est géré via session/cookie par défaut
- `/zh/` sur le site = URL du switcher langue (homepage chinoise), pas un préfixe de routing

**Pour des URLs `/zh/[slug]` (crawlables par Google) :**
- FaLang s'appuie sur le système natif Joomla (Language Filter plugin)
- Il faut des items de menu dédiés par langue (language = `zh-CN`, pas `*`)
- Language Filter génère alors les routes `/zh/[slug]` automatiquement
- FaLang traduit le contenu à ces routes
- Associations de langue entre les items FR et ZH recommandées

**État actuel du site :**
- Language Filter : activé, `remove_default_prefix: 1` (FR sans préfixe, ZH = `/zh/`)
- FaLang : 2153 traductions ZH en base
- Tous les items de menu en langue `*` → pas de routes `/zh/[slug]`
- hreflang `fr`, `zh`, `x-default` déjà dans le `<head>` via `mod_falang`

**Plan d'action pour URLs ZH :**
1. Identifier les pages stratégiques (cours, examens, accueil, contact)
2. Créer des items de menu en `zh-CN` pour ces pages
3. Créer les associations FR↔ZH dans Joomla
4. Language Filter génère les routes `/zh/[slug]`
5. Mettre à jour JSitemap (sources + hreflang)
→ Environ 20-30 items de menu à dupliquer pour les pages prioritaires
